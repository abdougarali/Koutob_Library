# إعداد Resend بسرعة (Quick Setup)

## الخطوات السريعة:

### 1. إنشاء حساب Resend
- اذهب إلى: https://resend.com
- سجّل حساب مجاني
- تحقق من بريدك الإلكتروني

### 2. الحصول على API Key
- اذهب إلى: https://resend.com/api-keys
- اضغط "Create API Key"
- امنحه اسماً (مثلاً: "Koutob Bookshop")
- انسخ المفتاح (يبدأ بـ `re_`)

### 3. إضافة المفاتيح إلى `.env.local`

افتح ملف `.env.local` وأضف:

```env
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxx
RESEND_FROM_EMAIL=onboarding@resend.dev
```

**ملاحظات مهمة:**
- استبدل `re_xxxxxxxxxxxxxxxxxxxxx` بمفتاحك الفعلي من Resend
- للاختبار في التطوير، استخدم `onboarding@resend.dev` (مُفعّل تلقائياً)
- للإنتاج، ستحتاج إلى التحقق من نطاقك الخاص

### 4. إعادة تشغيل السيرفر

بعد إضافة المفاتيح:
```bash
# أوقف السيرفر (Ctrl+C) ثم:
npm run dev
```

### 5. اختبار

1. اذهب إلى صفحة "نسيت كلمة المرور"
2. أدخل بريدك الإلكتروني
3. تحقق من صندوق الوارد (ورسائل البريد المزعج)

---

## استكشاف الأخطاء:

### الإيميل لا يصل؟
1. تحقق من أن `RESEND_API_KEY` موجود في `.env.local`
2. تحقق من أن السيرفر أعيد تشغيله بعد إضافة المفاتيح
3. تحقق من صندوق البريد المزعج (Spam)
4. تحقق من console للسيرفر - قد تكون هناك أخطاء

### في التطوير (Development):
- إذا لم يكن `RESEND_API_KEY` موجوداً، سيظهر الرابط في الـ console فقط
- هذا طبيعي ولا يعني وجود خطأ
